# Software Engineer Profile

**Domain:** Engineering

**Description:** LEVER interpretation for software engineers building production systems.

## Overview

The Software Engineer profile maps LEVER stages to traditional engineering career progression, from learning fundamentals to creating platforms that multiply engineering capability.

## Stage Interpretations

### Learn → Junior Engineer

**Description:** Learning fundamentals of software development.

**Evidence:**

- Completes coding tutorials and courses
- Understands basic CS concepts
- Learns team tools and practices
- Studies documentation and codebases

**Examples:**

- Completes language tutorials
- Sets up development environment
- Reads team coding standards
- Shadows senior engineers

---

### Execute → Engineer

**Description:** Implementing features and fixing bugs with guidance.

**Evidence:**

- Completes assigned tasks
- Writes code that passes review
- Debugs issues with support
- Contributes to team projects

**Examples:**

- Fixes bugs from issue tracker
- Implements well-specified features
- Writes unit tests
- Participates in code reviews

---

### Value → Senior Engineer

**Description:** Independently delivering significant technical work.

**Evidence:**

- Owns technical decisions
- Delivers complex features end-to-end
- Mentors junior engineers informally
- Trusted to work independently

**Examples:**

- Designs and implements new services
- Leads incident response
- Makes architectural decisions
- Improves team processes

---

### Enable → Staff Engineer

**Description:** Elevating team capability and technical direction.

**Evidence:**

- Sets technical direction for the team
- Mentors multiple engineers
- Leads technical initiatives
- Influences beyond immediate team

**Examples:**

- Defines team architecture
- Runs technical training sessions
- Leads cross-team projects
- Creates engineering standards

---

### Replicate → Principal/Architect

**Description:** Creating systems that multiply engineering capability.

**Evidence:**

- Creates reusable platforms and frameworks
- Defines engineering standards organization-wide
- Builds tools used across many teams
- Establishes technical communities

**Examples:**

- Builds internal developer platform
- Creates company-wide coding standards
- Develops shared libraries and tools
- Establishes architecture review process

## Career Progression

| Stage | Title | Scope |
|-------|-------|-------|
| Learn | Junior Engineer | Task |
| Execute | Engineer | Feature |
| Value | Senior Engineer | Project |
| Enable | Staff Engineer | Team |
| Replicate | Principal Engineer | Organization |

## Assessment Criteria

### Learn → Execute

- [ ] Demonstrates language proficiency
- [ ] Understands team codebase
- [ ] Can complete tasks with guidance
- [ ] Writes acceptable code

### Execute → Value

- [ ] Delivers features independently
- [ ] Makes sound technical decisions
- [ ] Handles ambiguous requirements
- [ ] Trusted with production systems

### Value → Enable

- [ ] Consistent technical excellence
- [ ] Actively develops others
- [ ] Leads technical initiatives
- [ ] Influences team direction

### Enable → Replicate

- [ ] Developed multiple senior engineers
- [ ] Created lasting technical improvements
- [ ] Sees systemic opportunities
- [ ] Can impact organization scale

## The IC vs Management Split

LEVER's Enable and Replicate stages can be reached on both IC and management tracks:

| Stage | IC Path | Management Path |
|-------|---------|-----------------|
| Enable | Staff Engineer | Engineering Manager |
| Replicate | Principal Engineer | Director/VP |

Both paths involve multiplication—through technical systems (IC) or through organizational systems (management).

## AI-Era Considerations

In the AI era, engineers can leverage AI to:

- **Accelerate Learn → Execute** with AI coding assistants
- **Reach Value faster** by automating routine tasks
- **Enable at scale** through AI-powered tools
- **Replicate through agents** that multiply capability

The core progression remains, but AI compression the timeline and amplifies impact at each stage.
